## 部署概述

### 1. 部署架构

#### 单机部署
```
[应用服务器]
  ├── UniAdmin 应用
  ├── SQL Server
  └── (可选) Redis
```

#### 分布式部署
```
[负载均衡器]
    ↓
[应用服务器 1] [应用服务器 2] [应用服务器 N]
    ↓              ↓              ↓
[共享数据库服务器]
```

### 2. 部署方式

- **独立部署**: 直接运行编译后的可执行文件
- **服务部署**: 作为 Windows 服务运行
- **Docker 部署**: 使用 Docker 容器化部署
- **云部署**: 部署到云平台（AWS、Azure、阿里云等）

## 环境准备

### 1. 硬件要求

#### 最低配置
- **CPU**: 2 核
- **内存**: 4GB
- **硬盘**: 20GB
- **网络**: 100Mbps

#### 推荐配置
- **CPU**: 4 核或更多
- **内存**: 8GB 或更多
- **硬盘**: 50GB SSD
- **网络**: 1Gbps

### 2. 软件要求

#### 操作系统
- Windows Server 2016 或更高版本
- Windows 10/11（开发环境）

#### 数据库
- SQL Server 2016 或更高版本
  - Express 版本（免费，适合小规模部署）
  - Standard/Enterprise 版本（生产环境推荐）

#### 可选组件
- Redis 5.0+（用于缓存和会话管理）
- IIS（可选，用于反向代理）
- Nginx（可选，用于反向代理）

### 3. 网络要求

#### 端口要求
- **应用端口**: 8077（默认，可配置）
- **数据库端口**: 1433（SQL Server 默认）
- **Redis 端口**: 6379（如果使用 Redis）

#### 防火墙配置
```bash
# Windows Firewall
netsh advfirewall firewall add rule name="UniAdmin" dir=in action=allow protocol=TCP localport=8077

# Linux Firewall
firewall-cmd --permanent --add-port=8077/tcp
firewall-cmd --reload
```

## 开发环境部署

### 1. 安装 Delphi

1. 下载 Delphi 12 Athens 安装包
2. 运行安装程序，选择典型安装
3. 安装完成后，启动 Delphi IDE

### 2. 安装 UniGUI

1. 从 FMSoft 官网下载 UniGUI 安装包
2. 运行安装程序，选择 Delphi 版本
3. 验证安装：File → New → Other → UniGUI for Delphi

### 3. 安装 SQL Server

#### SQL Server Express（免费）
1. 下载 SQL Server Express
2. 运行安装程序，选择基本安装
3. 设置 sa 密码
4. 启用 TCP/IP 协议：
   - 打开 SQL Server Configuration Manager
   - 展开 SQL Server 网络配置
   - 启用 TCP/IP
   - 重启 SQL Server 服务

### 4. 配置数据库

#### 创建数据库
```sql
-- 连接到 SQL Server
-- 服务器: localhost
-- 认证: SQL Server Authentication
-- 用户名: sa
-- 密码: your_password

-- 创建数据库
CREATE DATABASE UniAdminDB;
GO

USE UniAdminDB;
GO

-- 执行初始化脚本
-- 路径: database/init_schema.sql
```

#### 创建登录用户
```sql
-- 创建登录
CREATE LOGIN uniadmin_user WITH PASSWORD = 'strong_password';
GO

-- 创建用户
USE UniAdminDB;
CREATE USER uniadmin_user FOR LOGIN uniadmin_user;
GO

-- 授予权限
ALTER ROLE db_owner ADD MEMBER uniadmin_user;
GO
```

### 6. 编译和运行

## 生产环境部署

### 1. 编译发布版本

#### 编译配置
1. 打开 `UniAdmin.dproj`
2. Project → Options → Building → Delphi Compiler
3. 配置：
   - Configuration: Release
   - Target platform: Windows 64-bit
   - Optimization: On
   - Stack frames: Off
   - Debug information: None
   - Range checking: Off
   - Overflow checking: Off

#### 编译
1. Build → Build Project (Shift+F9)
2. 输出目录: `Win64\Release\`

```powershell
# 创建部署目录
New-Item -ItemType Directory -Path "deploy\UniAdmin"

New-Item -ItemType Directory -Path "deploy\UniAdmin\uploads"
```

### 3. 部署到服务器

#### 方式一: 手动部署
1. 将部署文件打包为 ZIP
2. 上传到目标服务器
3. 解压到目标目录（如 `C:\UniAdmin\`）
4. 配置 `ProjectConfig.json`
5. 运行 `UniAdmin.exe`

#### 方式二: 使用脚本部署
```powershell
# deploy.ps1

# 目标服务器信息
$server = "server.example.com"
$username = "admin"
$password = "password"
$targetPath = "C:\UniAdmin"

# 创建会话
$session = New-PSSession -ComputerName $server -Credential (New-Object System.Management.Automation.PSCredential($username, (ConvertTo-SecureString $password -AsPlainText -Force)))

# 复制文件
Copy-Item -ToSession $session -Path ".\deploy\UniAdmin" -Destination $targetPath -Recurse -Force

# 启动服务
Invoke-Command -Session $session -ScriptBlock {
    Start-Service -Name "UniAdmin"
}

# 关闭会话
Remove-PSSession $session
```

### 4. 配置为 Windows 服务

#### 使用 NSSM (Non-Sucking Service Manager)
1. 下载 NSSM: https://nssm.cc/download
2. 解压到 `C:\nssm\`

#### 安装服务
```powershell
# 安装服务
C:\nssm\nssm.exe install UniAdmin C:\UniAdmin\UniAdmin.exe

# 配置服务
C:\nssm\nssm.exe set UniAdmin AppDirectory C:\UniAdmin
C:\nssm\nssm.exe set UniAdmin DisplayName "UniAdmin Web Application"
C:\nssm\nssm.exe set UniAdmin Description "UniAdmin 企业级管理系统"
C:\nssm\nssm.exe set UniAdmin Start SERVICE_AUTO_START

# 启动服务
C:\nssm\nssm.exe start UniAdmin
```

#### 管理服务
```powershell
# 停止服务
C:\nssm\nssm.exe stop UniAdmin

# 重启服务
C:\nssm\nssm.exe restart UniAdmin

# 卸载服务
C:\nssm\nssm.exe remove UniAdmin
```

### 5. 配置反向代理

#### 使用 IIS
1. 安装 IIS 和 URL Rewrite 模块
2. 创建新站点
3. 配置反向代理规则：

```xml
<configuration>
  <system.webServer>
    <rewrite>
      <rules>
        <rule name="ReverseProxyInboundRule1" stopProcessing="true">
          <match url="(.*)" />
          <action type="Rewrite" url="http://localhost:8077/{R:1}" />
        </rule>
      </rules>
    </rewrite>
  </system.webServer>
</configuration>
```

#### 使用 Nginx
```nginx
server {
    listen 80;
    server_name uniadmin.example.com;

    location / {
        proxy_pass http://localhost:8077;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## Docker 部署

### 1. 创建 Dockerfile

```dockerfile
# 使用 Windows 基础镜像
FROM mcr.microsoft.com/windows/servercore:ltsc2022

# 安装必要组件
RUN powershell -Command \
    Install-WindowsFeature -Name Net-Framework-45-Core; \
    Install-WindowsFeature -Name Web-Server;

# 设置工作目录
WORKDIR /app

# 复制应用文件
COPY UniAdmin.exe /app/
COPY UniAdmin.cfg /app/
COPY ProjectConfig.json /app/
COPY metadata /app/metadata/
COPY plugins /app/plugins/

# 创建必要目录
RUN powershell -Command New-Item -ItemType Directory -Path /app/logs; \
    New-Item -ItemType Directory -Path /app/uploads

# 暴露端口
EXPOSE 8077

# 启动应用
CMD ["UniAdmin.exe"]
```

### 2. 创建 docker-compose.yml

```yaml
version: '3.8'

services:
  # 应用服务
  uniadmin:
    build: .
    container_name: uniadmin
    ports:
      - "8077:8077"
    volumes:
      - ./logs:/app/logs
      - ./uploads:/app/uploads
    depends_on:
      - db
      - redis
    environment:
      - DATABASE_SERVER=db
      - DATABASE_NAME=UniAdminDB
      - REDIS_SERVER=redis
    networks:
      - uniadmin-network

  # 数据库服务
  db:
    image: mcr.microsoft.com/mssql/server:2019-latest
    container_name: uniadmin-db
    environment:
      - ACCEPT_EULA=Y
      - SA_PASSWORD=YourStrong@Password
    ports:
      - "1433:1433"
    volumes:
      - db-data:/var/opt/mssql
    networks:
      - uniadmin-network

  # Redis 服务
  redis:
    image: redis:7-alpine
    container_name: uniadmin-redis
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data
    networks:
      - uniadmin-network

volumes:
  db-data:
  redis-data:

networks:
  uniadmin-network:
    driver: bridge
```

```bash
docker-compose up -d
```

#### 查看日志

## 负载均衡部署

### 1. 架构设计

```
[用户]
    ↓
[负载均衡器 / Nginx]
    ↓
    ├─→ [应用服务器 1:8077]
    ├─→ [应用服务器 2:8077]
    └─→ [应用服务器 3:8077]
            ↓
    [共享数据库/Redis]
```

### 2. 配置 Nginx 负载均衡

```nginx
upstream uniadmin_backend {
    # 负载均衡策略
    least_conn;

    # 应用服务器
    server app1.example.com:8077 weight=3;
    server app2.example.com:8077 weight=2;
    server app3.example.com:8077 weight=1;

    # 健康检查
    keepalive 32;
}

server {
    listen 80;
    server_name uniadmin.example.com;

    # 客户端请求体大小限制
    client_max_body_size 100M;

    # 超时设置
    proxy_connect_timeout 60s;
    proxy_send_timeout 60s;
    proxy_read_timeout 60s;

    location / {
        proxy_pass http://uniadmin_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket 支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";

        # 缓存配置
        proxy_cache_bypass $http_upgrade;
    }

    # 健康检查端点
    location /health {
        access_log off;
        return 200 "healthy\n";
        add_header Content-Type text/plain;
    }
}
```

## 监控与维护

### 1. 健康检查

#### 实现健康检查端点
```delphi
procedure TUniAdminServer.HandleHealthCheck(Request: TWebRequest;
  Response: TWebResponse);
begin
  try
    // 检查数据库连接
    if not FDConnection.Connected then
      raise Exception.Create('数据库未连接');

    // 检查 Redis 连接（如果启用）
    if RedisEnabled and not RedisClient.Connected then
      raise Exception.Create('Redis 未连接');

    // 返回健康状态
    Response.Content := '{"status":"healthy","timestamp":"' +
      ISO8601DateTime(Now) + '"}';
    Response.StatusCode := 200;
  except
    on E: Exception do
    begin
      Response.Content := '{"status":"unhealthy","error":"' +
        E.Message + '"}';
      Response.StatusCode := 503;
    end;
  end;
end;
```

```powershell
$timestamp = Get-Date -Format "yyyyMMdd_HHmmss"
$backupFile = "$backupPath\UniAdmin_$timestamp.bak"

# 创建备份目录
if (-not (Test-Path $backupPath)) {
    New-Item -ItemType Directory -Path $backupPath
}

# 执行备份
$sql = "BACKUP DATABASE $database TO DISK = '$backupFile' WITH FORMAT"
Invoke-Sqlcmd -Query $sql -ServerInstance localhost -Username sa -Password password

# 删除 7 天前的备份
Get-ChildItem $backupPath -Filter "*.bak" |
    Where-Object { $_.LastWriteTime -lt (Get-Date).AddDays(-7) } |
    Remove-Item
```

## 常见问题

### Q1: 应用无法启动

**检查步骤**:
1. 检查端口是否被占用
2. 检查数据库连接是否正常
3. 查看日志文件
4. 验证配置文件格式

**解决方案**:
