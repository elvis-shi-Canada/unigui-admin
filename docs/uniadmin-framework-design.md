## 概述

### 设计目标

创建一个类似 Django Admin 的零代码后台管理框架，实现：

- **插件化核心** - 框架以插件机制为核心，支持动态扩展
- **设计时可视化配置** - 拖拽组件、属性编辑器完成配置
- **运行时自动驱动** - 配置驱动 CRUD 逻辑，无需编写代码
- **高度可扩展** - 钩子方法 + 插件机制支持自定义业务逻辑
- **元数据驱动** - 数据库元数据 + 配置声明双重驱动

### 核心理念

```
插件化核心 + 设计时配置 → 运行时引擎 → 自动 CRUD
```

## 核心架构设计

### 插件化与模块化架构

#### 设计理念

插件化是 UniAdmin 的**核心设计**，而非扩展功能。框架从底层设计就支持：

- **插件机制** - 允许第三方开发者或团队以插件形式扩展功能（新控件类型、自定义操作按钮、报表引擎集成）
- **独立打包** - 插件可独立打包、动态加载，避免核心框架臃肿
- **模块化拆分** - SaaS、API、迁移工具等作为可选模块，核心保持轻量

#### 插件类型

| 插件类型 | 说明 | 应用场景 |
|---------|------|---------|
| **控件插件** | 自定义输入控件 | 富文本编辑器、代码编辑器、树形选择器、颜色选择器 |
| **操作插件** | 自定义操作按钮 | 批量邮件、生成报表、数据同步、审批提交 |
| **验证插件** | 自定义验证规则 | 身份证验证、手机号验证、银行卡验证、税号验证 |
| **导出插件** | 导出格式扩展 | Excel、PDF、Word、CSV、JSON |
| **导入插件** | 导入格式扩展 | Excel导入、CSV导入 |
| **图表插件** | 数据可视化 | 趋势图、饼图、柱状图、仪表盘 |
| **主题插件** | UI主题扩展 | 暗黑模式、扁平化、Material Design |

#### 模块类型

| 模块类型 | 说明 | 特点 |
|---------|------|------|
| **SaaS模块** | 多租户支持 | 租户隔离、独立配置、数据分离 |
| **API模块** | REST接口 | CRUD API、认证授权、版本管理 |
| **工作流模块** | 审批流程 | 状态机、流转规则、历史记录 |
| **通知模块** | 消息推送 | 邮件、短信、站内信、WebSocket |
| **迁移模块** | 数据库迁移 | 版本管理、脚本执行、回滚支持 |

#### 插件生命周期

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Loaded    │────→│  Installed  │────→│   Active    │
│  (已加载)   │     │  (已安装)   │     │   (激活)    │
└─────────────┘     └─────────────┘     └─────────────┘
       ↑                   │                   │
       │                   ↓                   ↓
       │            ┌─────────────┐     ┌─────────────┐

       │            │  (已禁用)   │     │  (停用)     │
       └────────────┴─────────────┘     └─────────────┘
```

#### 认证流程

```
用户登录请求
    ↓
验证用户名密码
    ↓
检查用户状态（是否禁用）
    ↓
检查密码策略（是否过期）
    ↓
生成会话 Token
    ↓
记录登录日志
    ↓
加载用户权限 → 跳转首页
```

#### 密码策略

| 策略 | 说明 |
|-----|------|
| 最小长度 | 密码最小字符数 |
| 复杂度 | 是否需要大小写、数字、特殊字符 |
| 有效期 | 密码有效天数（0表示永不过期） |
| 历史检查 | 不能与最近N次密码相同 |
| 登录失败锁定 | 连续失败N次后锁定 |

## 实现指南

### 开发工作流

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  1. 创建    │────→│  2. 设计    │────→│  3. 配置    │────→│  4. 运行    │
│    Frame    │     │    布局     │     │  ModelAdmin │     │    测试     │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
      ↓                   ↓                   ↓                   ↓
 继承BaseCrudFrame   放置面板和控件     设置属性和集合      框架自动构建
```

### 配置步骤

1. **创建 Frame** - 继承 TBaseCrudFrame
2. **设计布局** - 放置面板（分组容器）和输入控件
3. **配置 TUniAdminModel**
   - 设置 TableName、PrimaryKey、SoftDelete
   - 配置 ListDisplay（列表显示列）
   - 配置 ListFilter（过滤器）
   - 配置 SearchFields（搜索字段）
   - 配置 FieldSets（表单分组）
   - 配置 FieldControls（控件映射）
   - 配置 FieldLinks（字段联动）
   - 配置 Permissions（权限）
4. **运行测试** - 框架自动构建列表和表单

### 运行时流程

```
Initialize
    ↓
读取 TUniAdminModel 配置
    ↓
加载元数据（如有）→ 合并配置
    ↓
应用权限控制 → 过滤字段/按钮
    ↓
BuildGrid → 构建列表页
    ↓
BuildFilter → 构建过滤器
    ↓
BuildForm → 构建表单页
    ↓
CRUD 操作 → 通过钩子方法扩展
```
