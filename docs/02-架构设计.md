## 系统概述

### 1. 系统定位
UniAdmin 是一个基于 Delphi + UniGUI 的企业级 Web 管理系统框架，采用零代码开发理念，通过元数据驱动和插件化架构，实现快速构建企业管理应用。

### 2. 核心特性
- **零代码 CRUD**: 通过 TUniModelAdmin 组件实现自动化的增删改查
- **插件化架构**: 动态加载业务模块，支持热插拔
- **RBAC 权限系统**: 细粒度的角色和权限管理
- **元数据驱动**: 通过 JSON 配置定义业务逻辑
- **前后端分离**: UniGUI 自动处理前后端通信
- **多租户支持**: 支持多租户数据隔离

### 3. 系统目标
- **开发效率**: 比传统开发提升 5-10 倍
- **可维护性**: 模块化设计，降低耦合度
- **可扩展性**: 插件机制支持功能扩展
- **安全性**: 完善的权限控制和数据保护

## 数据架构

### 1. 数据库设计原则

- **规范化**: 遵循第三范式（3NF）
- **索引优化**: 为常用查询字段创建索引
- **数据完整性**: 使用外键约束
- **审计追踪**: 记录数据变更历史
- **软删除**: 使用标记删除而非物理删除

### 2. 核心表结构

#### 用户表 (Users)
```sql
CREATE TABLE Users (
    Id INT IDENTITY(1,1) PRIMARY KEY,
    UserName NVARCHAR(50) UNIQUE NOT NULL,
    Password NVARCHAR(255) NOT NULL,
    Email NVARCHAR(100) UNIQUE,
    FirstName NVARCHAR(50),
    LastName NVARCHAR(50),
    IsActive BIT DEFAULT 1,
    CreatedDate DATETIME DEFAULT GETDATE(),
    UpdatedDate DATETIME,
    LastLoginDate DATETIME,
    TenantId INT NULL,  -- 多租户支持
    CONSTRAINT CHK_Email CHECK (Email LIKE '%@%.%')
);

CREATE INDEX IX_Users_UserName ON Users(UserName);
CREATE INDEX IX_Users_Email ON Users(Email);
CREATE INDEX IX_Users_TenantId ON Users(TenantId);
```

#### 角色表 (Roles)
```sql
CREATE TABLE Roles (
    Id INT IDENTITY(1,1) PRIMARY KEY,
    Name NVARCHAR(50) UNIQUE NOT NULL,
    Description NVARCHAR(200),
    IsSystem BIT DEFAULT 0,  -- 系统角色不可删除
    CreatedDate DATETIME DEFAULT GETDATE(),
    TenantId INT NULL
);

CREATE INDEX IX_Roles_Name ON Roles(Name);
```

#### 权限表 (Permissions)
```sql
CREATE TABLE Permissions (
    Id INT IDENTITY(1,1) PRIMARY KEY,
    Code NVARCHAR(100) UNIQUE NOT NULL,
    Name NVARCHAR(100) NOT NULL,
    Description NVARCHAR(200),
    Module NVARCHAR(50),
    Category NVARCHAR(50)
);

CREATE INDEX IX_Permissions_Code ON Permissions(Code);
CREATE INDEX IX_Permissions_Module ON Permissions(Module);
```

#### 角色权限关联表 (RolePermissions)
```sql
CREATE TABLE RolePermissions (
    RoleId INT NOT NULL,
    PermissionId INT NOT NULL,
    PRIMARY KEY (RoleId, PermissionId),
    FOREIGN KEY (RoleId) REFERENCES Roles(Id) ON DELETE CASCADE,
    FOREIGN KEY (PermissionId) REFERENCES Permissions(Id) ON DELETE CASCADE
);
```

#### 用户角色关联表 (UserRoles)
```sql
CREATE TABLE UserRoles (
    UserId INT NOT NULL,
    RoleId INT NOT NULL,
    PRIMARY KEY (UserId, RoleId),
    FOREIGN KEY (UserId) REFERENCES Users(Id) ON DELETE CASCADE,
    FOREIGN KEY (RoleId) REFERENCES Roles(Id) ON DELETE CASCADE
);
```

#### 审计日志表 (AuditLogs)
```sql
CREATE TABLE AuditLogs (
    Id BIGINT IDENTITY(1,1) PRIMARY KEY KEY,
    UserId INT NULL,
    UserName NVARCHAR(50),
    Action NVARCHAR(50) NOT NULL,  -- INSERT, UPDATE, DELETE
    TableName NVARCHAR(100) NOT NULL,
    RecordId INT NOT NULL,
    OldValue NVARCHAR(MAX) NULL,
    NewValue NVARCHAR(MAX) NULL,
    IpAddress NVARCHAR(50),
    CreatedDate DATETIME DEFAULT GETDATE()
);

CREATE INDEX IX_AuditLogs_UserId ON AuditLogs(UserId);
CREATE INDEX IX_AuditLogs_TableName ON AuditLogs(TableName);
CREATE INDEX IX_AuditLogs_CreatedDate ON AuditLogs(CreatedDate);
```

### 3. 安全策略

#### 密码策略
- 最小长度: 8 位
- 必须包含大小写字母
- 必须包含数字
- 必须包含特殊字符
- 使用 BCrypt 加密存储

#### 会话策略
- 会话超时: 30 分钟
- 单点登录: 支持
- 会话固定攻击防护: 每次登录重新生成 Session ID
- 并发登录限制: 可配置

#### 输入验证
- 所有用户输入必须验证
- 使用白名单而非黑名单
- 防止 XSS 攻击
- 防止 CSRF 攻击

#### SQL 注入防护
- 使用参数化查询
- 禁止拼接 SQL 语句
- 使用 ORM 或数据访问层
- 最小权限原则

## 设计模式

### 1. 使用的设计模式

#### 单例模式 (Singleton)
- **应用场景**: 配置管理器、日志管理器
- **示例**: `TConfigManager.Instance`

#### 工厂模式 (Factory)
- **应用场景**: 插件创建、表单创建
- **示例**: `TFormFactory.CreateForm('UserForm')`

#### 观察者模式 (Observer)
- **应用场景**: 事件处理、数据变更通知
- **示例**: `TEventBus`

#### 策略模式 (Strategy)
- **应用场景**: 权限验证策略、缓存策略
- **示例**: `IPermissionStrategy`

#### 仓储模式 (Repository)
- **应用场景**: 数据访问
- **示例**: `IUserRepository`

#### 依赖注入 (Dependency Injection)
- **应用场景**: 服务注入、模块注入
- **示例**: `ServiceContainer.Resolve`

### 2. 架构模式

#### MVC 模式
- **Model**: 数据模型和业务逻辑
- **View**: UniGUI 表单和组件
- **Controller**: 事件处理和数据绑定

#### 分层架构
- **表现层**: UniGUI 表单
- **业务逻辑层**: Services
- **数据访问层**: Repository
